## 1. 왜 냉각이 필요한가?

**물리학의 에너지 보존 법칙**: 서버에 공급한 전기 에너지는 사라지지 않고, **전부 열로 변환**됩니다.

```
서버에 전기 1kWh 공급
   → CPU/메모리/SSD가 연산
   → 연산 결과로 생긴 열이 서버실 공기를 데움
   → 제거하지 않으면 온도가 계속 상승 → 서버 고장
```

따라서 **냉각 시스템이 제거해야 할 열량 ≈ IT 장비 전력 소비량**입니다.

> 예: 서버 500대가 6,200 kW를 소비하면, 냉각 시스템도 최소 6,200 kW 분량의 열을 제거해야 합니다.

## 2. 서버 전력 소비 모델 (SPECpower)

### 개념

서버는 CPU를 100% 사용할 때 가장 많은 전력을 소비하고, 아무 작업도 없을 때도 **유휴 전력**을 소비합니다.

### 수식

```
P_server = P_idle + (P_max - P_idle) × cpu_utilization
```

| 기호 | 의미 | 단위 |
|------|------|------|
| `P_server` | 서버 전력 소비량 | Watt (W) |
| `P_idle` | 유휴 상태 전력 (CPU 0% 사용) | W |
| `P_max` | 최대 전력 (CPU 100% 사용) | W |
| `cpu_utilization` | CPU 사용률 | 0.0 ~ 1.0 |

### 참조값 (SPECpower_ssj2008 기준)

| 서버 종류 | P_idle | P_max | 비고 |
|-----------|--------|-------|------|
| CPU 서버 (Intel Xeon) | 200 W | 500 W | 일반 서버 |
| GPU 서버 (A100 × 4장) | 300 W | 1,500 W | AI 학습 서버 |

### 예시 계산

```
CPU 서버 500대, 평균 CPU 사용률 70%:
P_server 1대 = 200 + (500 - 200) × 0.7 = 200 + 210 = 410 W
총 IT 전력 = 410 W × 500대 = 205,000 W = 205 kW
```

## 3. 냉각 부하 계산

### 개념

차가운 공기가 서버를 통과하면서 열을 흡수해 뜨거워집니다. 이때 공기가 흡수한 열량이 **냉각 부하**입니다.

```
[차가운 공기 투입] → [서버 통과] → [뜨거운 공기 배출]
   공급 온도(T_supply)              환기 온도(T_return)
```

### 수식

```
Q_cooling = ṁ × c_p × ΔT
```

| 기호 | 의미 | 단위 | 값 |
|------|------|------|----|
| `Q_cooling` | 냉각 부하 (제거해야 할 열량) | kW (= kJ/s) | 계산값 |
| `ṁ` | 공기 질량 유량 (초당 지나가는 공기 무게) | kg/s | 운영 설정 |
| `c_p` | 공기 비열 (공기 1kg을 1°C 올리는 데 필요한 에너지) | kJ/(kg·K) | **1.005** (고정값) |
| `ΔT` | 온도 차이 = 환기 온도 - 공급 온도 | °C | 운영 측정값 |

### 직관적 이해

> **c_p = 1.005 kJ/kg·K** 이란?
> 공기 1 kg을 1°C 높이는 데 약 1 kJ의 에너지가 필요하다는 의미입니다.
> 물(4.18 kJ/kg·K)보다 훨씬 작아서, 공기는 물보다 열을 4배 적게 저장합니다.

### 예시 계산

```
공기 유량: 50 kg/s
공급 온도: 18°C (서버실에 투입되는 차가운 공기)
환기 온도: 27°C (서버를 통과한 뜨거운 공기)
ΔT = 27 - 18 = 9°C

Q_cooling = 50 × 1.005 × 9 = 452.25 kW
```

## 4. 칠러 효율 모델 (COP)

### 개념

칠러(Chiller)는 에어컨의 대형 버전입니다. 전기를 소비해서 냉각수를 차갑게 만듭니다.

**COP(Coefficient Of Performance, 성능계수)**: 전기 1 kW를 써서 몇 kW의 열을 제거할 수 있는지를 나타냅니다.

```
COP = 제거한 열량(kW) / 소비한 전력(kW)
```

- COP = 6: 전기 1 kW로 열 6 kW 제거 (좋은 상태)
- COP = 2: 전기 1 kW로 열 2 kW 제거 (더운 날)

### 수식 — COP와 외기 온도 관계

```
COP = max(2.0,  6.0 - 0.1 × (T_outdoor - 15))
```

| 기호 | 의미 |
|------|------|
| `T_outdoor` | 외기 온도 (°C) |
| `6.0` | 기준 COP (외기 15°C 기준) |
| `0.1` | 온도 1°C 상승당 COP 저하량 |
| `max(2.0, ...)` | 아무리 더워도 COP는 최소 2.0 보장 |

**왜 외기가 더우면 COP가 낮아지나?**
에어컨과 같은 원리입니다. 바깥이 더울수록 열을 내보내기 어려워지므로 같은 냉각 효과를 내기 위해 더 많은 전력이 필요합니다.

### 수식 — 칠러 전력

```
P_chiller = Q_cooling / COP
```

### 예시 계산

| 상황 | 외기 온도 | COP | 냉각 부하 | 칠러 전력 |
|------|-----------|-----|-----------|-----------|
| 봄/가을 쾌적 | 15°C | 6.0 | 500 kW | 83 kW |
| 여름 더운 날 | 35°C | 4.0 | 500 kW | 125 kW |
| 폭염 | 38°C | 3.7 | 500 kW | 135 kW |

## 5. 자연공조 (Free Cooling)

### 개념

외기가 충분히 차가울 때, **칠러 없이** 외부 공기(또는 냉각탑 물)로 직접 서버실을 냉각하는 방식입니다.

전기 에너지 절감 효과가 매우 큽니다. 팬 전력만 소비하므로 칠러 대비 약 95% 전력 절감 가능.

### 냉각 모드 전환 기준

```
외기 온도 < 15°C  → 완전 자연공조 (Full Free Cooling)
15°C ≤ 외기 < 22°C → 혼합 냉각 (Hybrid: 자연공조 + 칠러 보조)
외기 온도 ≥ 22°C  → 기계식 냉방 (Chiller Only)
```

### 자연공조 효율 수식

```
efficiency =
  1.0                                           (T_outdoor < 15°C)
  1 - (T_outdoor - 15) / (22 - 15)             (15°C ≤ T < 22°C)
  0.0                                           (T_outdoor ≥ 22°C)
```

습도 보정 (높은 습도는 효율 소폭 감소):
```
humidity_factor = 1.0 - max(0, (humidity% - 50) / 500)
```

### 혼합 냉각 전력

```
P_cooling = (Q_cooling × chiller_fraction) / COP
          + (Q_cooling × free_cooling_fraction × 0.05)
```
> 자연공조 구간에서는 팬 전력만 발생 (냉각 부하의 약 5%)

## 6. PUE — 데이터센터 효율 지표

### 개념

**PUE(Power Usage Effectiveness)**: 데이터센터에서 IT 장비 외에 얼마나 많은 전력을 쓰는지 나타냅니다.

```
PUE = 데이터센터 총 전력 / IT 장비 전력
    = (P_IT + P_냉각 + P_기타) / P_IT
```

| 기호 | 의미 | 비고 |
|------|------|------|
| `P_IT` | IT 장비 전력 | 서버, 스토리지, 네트워크 |
| `P_냉각` | 냉각 시스템 전력 | 칠러, 팬, 냉각탑 |
| `P_기타` | 기타 전력 | 조명, UPS 손실, 배전 손실 (IT 전력의 약 5%) |

### PUE 해석

```
PUE = 1.0 → 이상적 (냉각·기타 전력 = 0)
PUE = 1.5 → IT 전력 외에 50% 추가 소비
PUE = 2.0 → IT 전력과 동일한 양을 냉각 등에 추가 소비 (비효율적)
```

### 벤치마크

| 데이터센터 | PUE | 평가 |
|-----------|-----|------|
| NAVER 각 춘천 | **1.09** | 세계 최고 수준 (목표) |
| Google 글로벌 | 1.10 | 세계 최상급 |
| 그린DC 인증 기준 | 1.66 | 국내 최소 효율 기준 |
| 글로벌 평균 | 1.58 | 평균 수준 |
| 국내 민간 평균 | 2.03 | 비효율 |

### 예시 계산

```
IT 전력:   6,200 kW
냉각 전력:   900 kW
기타 전력:   310 kW (IT의 5%)
총 전력:   7,410 kW

PUE = 7,410 / 6,200 = 1.195
```

## 7. ESG 지표 — 탄소·물 사용량

### 탄소 배출량

```
탄소 배출량 (tCO2) = 전력 소비량 (MWh) × 0.459 (tCO2/MWh)
```

> 0.459 tCO2/MWh: 국내 전력 탄소 배출계수 (한국 전력 믹스 기준)

### WUE (Water Usage Effectiveness)

```
WUE = 물 사용량 (L) / IT 전력 소비량 (kWh)
```

| 냉각 방식 | WUE | 비고 |
|-----------|-----|------|
| 냉각탑(증발 냉각) | 1.8 L/kWh | 가장 물을 많이 씀 |
| 자연공조 | 0.2 L/kWh | 거의 물 미사용 |
| 공냉식 | 0.0 L/kWh | 물 전혀 미사용 |

## 8. 수식 전체 요약

```python
# 1. 서버 전력 (SPECpower)
P_server_W = P_idle + (P_max - P_idle) * cpu_utilization
P_IT_kW    = P_server_W * num_servers / 1000

# 2. 냉각 부하 (에너지 보존 법칙)
Q_cooling_kW = P_IT_kW  # 정상 상태: IT 전력 전부가 열로 변환

# 또는 공기 유량으로 계산:
Q_cooling_kW = m_dot * c_p * (T_return - T_supply)
#              (kg/s) × (kJ/kg·K) × (°C) = kW

# 3. 칠러 COP와 전력
COP          = max(2.0, 6.0 - 0.1 * (T_outdoor - 15))
P_chiller_kW = Q_cooling_kW / COP

# 4. 자연공조 가능 조건
free_cooling_available = T_outdoor < 15  # 완전 자연공조
# (15 ≤ T < 22: 혼합 냉각, T ≥ 22: 기계식 전용)

# 5. PUE
P_other_kW = P_IT_kW * 0.05  # 기타 전력 (5%)
P_total_kW = P_IT_kW + P_chiller_kW + P_other_kW
PUE        = P_total_kW / P_IT_kW

# 6. 탄소 배출량
carbon_tco2 = (P_total_kW * hours / 1000) * 0.459
```
